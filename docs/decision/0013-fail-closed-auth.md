# ADR-0013:認証をfail-closedにする

## 状況

既定で公開されるダッシュボードは,設定漏れがそのまま情報漏えいになる

## 決定

認証が設定されるまでダッシュボードもREST APIも生えず404を返す
同梱のBearerとCloudflare Access JWT検証を用意し,任意のHonoミドルウェアに差し替え可能にする

認証が設定されている場合に限り,ダッシュボードのHTMLだけは未認証でも返す
HTMLはアプリケーションコードでありデータを含まない
これを閉じるとブラウザがトークンを入力する手段を持てず, DevToolsでcookieを手打ちさせることになる

## 帰結

設定漏れが「動かない」として現れる方が静かな公開より安全

HTMLを開けることでTsumugiの導入自体は外から分かるが,ジョブの内容も件数も漏れない
APIは常に保護され,未認証のAPIアクセスは401を返す
