# ADR-0016:明細はD1,時系列はAnalytics Engineに置く

## 状況

sweepで明細を消すとメトリクスも一緒に消える
明細とメトリクスは要求される保持期間が違う

## 決定

個別ジョブの明細と一覧はD1,時系列メトリクスはAnalytics Engineに書く

## 帰結

sweepがどれだけ攻撃的でもメトリクスは残る
バインディングが1つ増えるがセットアップの負担は軽い

## メトリクスの配送保証(#7)

`writeDataPoint`は冪等でなく, 冪等な投影と同じ再試行単位に置くと再試行で件数と所要時間が二重に加算される
投影のカーソルを先に進めてからメトリクスを書き, 非冪等な書き込みを再試行単位から外す
この便で失敗したメトリクスは載らずat-most-onceになる, 二重加算を避ける方を優先する
メトリクスは省略可能な機能なので, 書き出しの失敗はジョブ調停を含むtick全体を止めない
